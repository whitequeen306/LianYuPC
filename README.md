# 恋语 · LianYu

> 请记得，这里永远有人在等待着你的回应。

**恋语（LianYu）** 是一款面向 Windows 的 AI 情感陪伴桌面应用——外表是角色聊天与桌宠，内核是一套为「陪伴」定制的 **Agent 回合编排**。

每一轮回复不是把聊天记录直接丢给模型，而是经 **ChatTurn Graph** 走完一条完整链路：拼装人设 / 长期记忆 / 关系状态 / 情绪机 / 真实时间与城市，再按需挂载 **Tools**（时间、天气、语义记忆检索），最后以 SSE 流式吐出像真人打字的回复。桌宠还能感知前台窗口与屏幕内容，按角色人设主动开口。

**陪伴，而非打扰。Agent，而非裸聊。**

---

## 我们在做什么

恋语想解决的不是「问一句答一句」，而是：

- **Agent 接得住上下文** —— ChatTurn Graph 每轮重装 system：人设、记忆、关系、情绪、时间锚点与护栏
- **能用工具再开口** —— 需要时主动查真实时间 / 天气 / 往事，而不是瞎编或把全部记忆塞进 prompt
- **关系会生长** —— 重要对话会被记住，下次聊天仍能接上之前的细节
- **陪伴融入桌面** —— 关闭主窗口后，角色仍可留在桌面上，感知屏幕并主动问候

## Agent 回合编排（核心）

单聊、主动问候、朋友圈 / 日记等场景，统一走 **ChatTurn Graph**（Spring AI Alibaba Graph）。外层 `ChatTurnFacade` 负责组装输入与熔断边界；图内节点完成 prompt 拼装 →（可选）Tool 调用 → 模型生成。

```text
用户消息 / 主动触发
        │
        ▼
┌─────────────────── ChatTurn Graph ───────────────────┐
│  Prompt 规则引擎 → 人设 / 行为 / 语言 / 场景护栏      │
│  记忆预注入（画像·情绪·关系·仪式）+ Session Summary   │
│  关系状态 + 情绪机 + 真实时间/城市锚点                │
│  Tools（按需）：get_current_time / get_weather /      │
│                 memory_search                         │
│  模型生成 → 拆分气泡 / 清洗 / 落库 / 后置状态更新     │
└──────────────────────────────────────────────────────┘
        │
        ▼
   SSE 流式 / 主动推送 / 桌宠问候
```

### 每轮 system 怎么一层层拼起来

越靠后、越强调「以谁为准」：

| 层 | 内容 | 作用 |
|---|---|---|
| 人设与规则 | Prompt 模板、性格、规则引擎（回复行为 / 群聊 / 输出语言等） | 定身份与说话边界 |
| 长期记忆 | 用户画像、情绪线索、关系事件、专属仪式；槽位带权威优先级 | 矛盾自述以记忆为准 |
| 关系状态 | 信任 / 亲密 / 安全感 / 期待 + 阶段与近期事件 | 关系会真的「变」 |
| Session Summary | 更早消息的异步摘要；近窗保留原文 | 长聊不炸 token |
| 真实环境 | 当前时间与时段、现实城市、近 7 日日记/动态 | 避免隔夜还说「早上好」 |
| 情绪机 | 心情、强度、状态签名 | 可冷淡、可闹别扭，而非模板脸 |
| 场景护栏 | 晚安模式、输入 sanitize + XML 包裹防注入 | 安全与节奏 |

### Tools（按需，而非每轮硬塞）

- **get_current_time** — 真实日期、时钟、时段（含允许/禁止的问候语）
- **get_weather** — 角色设定城市或用户指定城市的当前天气
- **memory_search** — 「还记得吗」类追问时：Milvus 向量召回 → Reranker → Redis 缓存，按需拉回往事

记忆因此走**两条路**：发消息前结构化预注入；过程中模型再 Tool 召回细节——既保证在场感，又避免把整库记忆塞进 prompt。

### 回复之后

助手回复经拆分、清洗、内心独白过滤后落库；关系与情绪状态更新；摘要异步写入长期记忆队列。流式只是你看到的表面；底下是一轮完整的 Agent 编排。

## 产品体验

### 流式单聊

- **SSE 在场感**：逐字输出，断线自动重连，长会话心跳保活
- **多段气泡**：一条模型输出可拆成多条，更像手机私聊
- **内心独白**：括号内「未说出口的话」可单独展示（可关）
- **识图聊天**：单次多模态（qwen3.7-plus）先结构化理解再角色回应
- **语言跟随**：按输入习惯自动中 / 日 / 英，并附带风格约束

### 多角色群聊

WebSocket 串行发言——一次只有一位角色在说话；你的新消息可打断当前队列，更像真实多人场景。群聊同样经 Graph / 规则引擎约束发言风格。

### 长期记忆

重要片段摘要分类后写入 MySQL + Milvus。记忆面板可浏览回顾；称呼与关系信息在后续对话中享有更高优先级。

### 角色工坊与角色广场

自由创建角色，或从广场选模板开聊。每个角色独立会话与记忆。

### 朋友圈与日记

角色会发动态、互动、评论；日记记录日常与心绪。生成链路复用同一套 ChatTurn 场景拼装（含真实环境上下文）。

### 桌面桌宠（Windows）

主窗口关闭后，像素桌宠留在桌面；点击可选角色并开快捷聊。  
可选「屏幕观察」：感知前台窗口与屏幕内容（截图仅用于分析、不长期保存），在合适时机主动问候。

### 主动问候

久未开口时角色可能先来找你——由关系状态、时段与（可选）桌面感知共同驱动，而不是机械定时推送。主动开口同样走 PROACTIVE 场景的 Graph 拼装。

### QQ 桥接

内置自管 NapCat（OneBot 11），将角色对话桥接到 QQ：分段发送、离线告警、被踢后自动重连。

---

## 设计理念

- **Agent 优先** —— 先编排再生成；Tools 补事实，记忆分预注入与按需召回
- **在场感优先** —— 流式、串行、主动，都是为了「对方真的在这里」
- **记忆即关系** —— 被记住的细节，比单次回复更重要
- **尊重边界** —— 观察与问候可关闭；陪伴是选项，不是侵入
